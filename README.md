# 批量图片转文字（batch-image-to-text）

一个本地批量 OCR 工具：选择图片文件夹后，逐张提取图片原文，并生成可复核的 Markdown、JSONL 和 CSV 结果。

这个上传版本包含本地 `runtime/` 运行环境，可在 Windows 上双击启动。

## 解决的问题

在整理截图知识库时，直接让 AI 看图归类容易丢失原文、漏掉书名、推荐名单、观点细节。本工具把流程拆成两步：

1. 先逐图提取原文，保留文件名、路径、状态和置信度。
2. 再基于原文做清洗、拆分、归类和知识库写入。

## 当前 Demo 能力

- 选择一个图片文件夹。
- 可选择是否包含子文件夹。
- 可选择在输出文件夹里新建一个自命名子文件夹，再输出结果。
- 处理时显示进度条、已完成数量、剩余数量和当前文件名。
- 批量处理 `.jpg`、`.jpeg`、`.png`、`.bmp`、`.webp`、`.tif`、`.tiff`。
- 每张图单独输出，不把多张图混成一个主题。
- 生成低置信度、空结果、失败图片的复核清单。
- 支持命令行和简单 GUI 两种入口。

## 安装依赖

如果不使用内置 `runtime/`，也可以自行安装依赖：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

## 运行

推荐双击：

```text
启动工具.vbs
```

这个入口不会显示黑色终端窗口，只显示工具页面。

如果 `.vbs` 被系统拦截，可以用备用入口：

```text
启动工具.bat
```

命令行：

```powershell
python -m src.app --input "C:\Pictures\input" --output "C:\Pictures\ocr-output"
```

## 界面选项说明

- `包含子文件夹`：勾选后，会连所选图片文件夹里的子文件夹一起处理；不勾选只处理当前文件夹里的图片。
- `在输出文件夹中新建子文件夹`：可选。勾选后需要填写新文件夹名称，结果会输出到“输出文件夹 / 新文件夹名称”。

## 输出文件

- `ocr_raw.md`：按图片顺序保存原文，适合直接阅读。
- `ocr_raw.jsonl`：一张图片一条结构化记录，适合后续程序清洗。
- `images_index.csv`：图片路径、状态、字数、置信度总览。
- `ocr_review.md`：需要人工复核的图片清单。

## 已知限制

- OCR 只能尽量提取原文，不能保证 100% 正确。
- 低清、长截图、小字、遮挡、评论区密集文字可能需要人工复核。
- 当前版本只做原文提取，不做知识点归类，也不写入知识库。
- 当前默认使用 RapidOCR + ONNXRuntime，比上一版 PaddleOCR 后端快很多。
- OCR 速度和图片尺寸、文字密度有关，长截图仍会比普通图片慢。

## 安全与隐私

工具默认只在本地读取图片并写出结果，不上传图片。请不要把包含私人信息的输出文件公开提交到 GitHub。
