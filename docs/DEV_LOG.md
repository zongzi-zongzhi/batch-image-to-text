# Dev Log

## 2026-06-03

### 完成内容

- 创建项目结构。
- 明确 MVP：选择文件夹、批量 OCR、输出原文和复核清单。
- 增加 GUI 与命令行入口。
- 增加 GUI 启动脚本。
- 将项目改为自包含工具包：内置 Python 运行环境、OCR 依赖和 OCR 模型。
- 将项目文件夹改名为 `batch-image-to-text`，避免中文路径导致 PaddleOCR 模型加载兼容问题。
- 禁用 PaddleOCR 的 MKLDNN/oneDNN 加速，避免 Windows CPU 推理错误。
- 默认使用 PP-OCRv5 mobile 模型，减少本地批量识别的耗时和模型体积。
- GUI 增加“包含子文件夹”的说明文字。
- GUI 增加可选的“在输出文件夹中新建子文件夹”功能。
- GUI 增加 Windows DPI awareness 设置，改善界面发糊问题。
- GUI 增加实时进度条、已完成/剩余数量和当前文件名显示。
- 增加 `启动工具.vbs` 无终端启动入口，减少命令行窗口干扰。
- 默认 OCR 后端从 PaddleOCR 切换为 RapidOCR + ONNXRuntime；同一张测试截图从几十秒降到约 6 秒。
- 静默 OCR 引擎日志，减少控制台输出干扰。

### 决策原因

用户最在意原文不丢失，因此第一版不做知识点整理，只做逐图原文提取。

### 遗留问题

- 尚未接入高精度多模态复核。
- 长截图切分仍属于后续能力。

### 下一步

- 用真实图片文件夹进行试跑。
- 根据 OCR 效果决定是否增加二次复核模式。
