# PRD：批量图片转文字

## 项目背景

用户有大量截图需要整理进个人知识库。直接从图片归纳知识点会导致原文丢失，尤其容易漏掉小说书籍推荐、评论区观点、作者名和细节表达。

## 项目目标

建立一个本地工具，先把图片中的文字完整提取出来，再把原文交给后续清洗和知识库归类流程。

## 目标用户

- 需要整理截图知识库的个人用户。
- 需要批量提取中文截图原文的人。

## 用户场景

用户选择一个图片文件夹，工具自动遍历其中图片，逐张提取文字，并输出原文记录和复核清单。

## 输入内容

- 图片文件夹路径。
- 输出文件夹路径。

## 输出结果

- 按图片拆分的原文 Markdown。
- 每张图片一条记录的 JSONL。
- 图片处理索引 CSV。
- 需要人工复核的图片清单。

## 核心功能

1. 批量读取图片。
2. 调用 OCR 引擎提取文字。
3. 保留每张图的路径、文件名、状态、置信度和原文。
4. 标注空结果、失败和低置信度图片。
5. 导出多种结果文件，供后续清洗使用。

## 用户使用流程

1. 打开工具。
2. 选择图片文件夹。
3. 选择输出文件夹。
4. 点击开始提取。
5. 查看 `ocr_raw.md` 和 `ocr_review.md`。

## MVP 范围

- 本地运行。
- 支持常见图片格式。
- 支持 GUI 和命令行。
- 使用 PaddleOCR 作为第一版 OCR 引擎。
- 不做知识库写入。

## 非目标范围

- 不自动创建或修改知识库 Markdown。
- 不自动总结观点。
- 不自动判断合并/新建。
- 不上传图片到外部服务。

## 后续扩展方向

- 增加多模态 API 高精度复核模式。
- 增加长截图切分。
- 增加书名、作者、推荐名单的二次抽取。
- 增加结果对照预览界面。

## 成功标准

- 能对指定文件夹内图片批量输出原文。
- 每张图片都有独立记录。
- 失败和不确定结果能被明确列入复核清单。
